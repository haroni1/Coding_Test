## 💡 문제 이해
> 문제를 나만의 언어로 요약해서 적어보세요.

- **문제**: 정수 두 개를 입력받아서 형식에 맞게 출력하는 문제

- **핵심 조건:** 
			- 입력은 **한 줄에 두 개의 숫자가 공백으로 주어진다**
    
			- 출력은 **정해진 형식(a = 값, b = 값)** 그대로 나와야 한다
    
			- 범위는 `-100000 ~ 100000` (하지만 연산은 없음 → 출력만 수행)

---

## 🚀 접근 방식 (Logic)
> 코드를 짜기 전, 어떤 알고리즘과 자료구조를 쓸지 설계하는 단계입니다.

1. **입력받기:** 
	- C#에서는 `Console.ReadLine()`으로 한 줄 전체를 읽는다.
    
	- 입력은 문자열로 들어오기 때문에 바로 숫자가 아니라 `"4 5"` 같은 문자열이다.

2. **입력 파싱**:** 
	- 숫자가 공백으로 구분되므로 `.Split()`을 사용해 `"4"` `"5"`로 나눈다.
    
	- 이후 `int.Parse()` or `Convert.ToInt32()`로 정수로 변환한다.
    
	- 이유: 지금은 출력만 하지만, **추후 계산이 필요할 수 있기 때문**.
	

3. **출력:** 
	- 형식을 그대로 지켜 `"a = 숫자"`, `"b = 숫자"` 출력한다.
    
	- 문자열 보간($"") 문법 사용하면 더 깔끔하다.
4. **시간 복잡도:** 
	 * 단순 입력 2개 처리 > O(1)

## 💻 정답 코드 (C#)

``` C#
using System;

class Program
{
    static void Main()
    {
        string[] input = Console.ReadLine().Split();

        int a = int.Parse(input[0]);
        int b = int.Parse(input[1]);

        Console.WriteLine($"a = {a}");
        Console.WriteLine($"b = {b}");
    }
}
```

## 📝 오답 노트 & 회고
> 틀렸다면 왜 틀렸는지, 맞았다면 더 좋은 방법은 없는지 기록하세요.

- **틀린 이유:** - 
	- 입력이 문자열이기 때문에 **그대로 숫자로 쓰면 안 된다**
    
	- `input[0]`이 숫자가 아니라 `"4"`라는 **문자열(string)** 이라는 점을 주의해야 함
    
	- `.Split()`을 쓰지 않으면 `"4 5"`가 `"['4',' ','5']"` 형태가 되어 오작동 가능

- **배운 점:** 
	- - 콘솔 입력은 기본적으로 **문자열로 들어온다**
    
	- 숫자 연산 가능성을 고려하면 **int로 변환하는 습관이 중요**
    
	- 문자열 입력 → Split → Parse → 출력이라는 입력 처리 흐름을 이해함

- **다른 방식 참고:** 
	- 문자열 붙은 입력 `"45"`라면 `.Split()` 없이 `char` 인덱싱으로 가능하지만 
     공백 입력이 더 일반적이므로 `.Split()`이 더 안정적이고 범용적이다.
