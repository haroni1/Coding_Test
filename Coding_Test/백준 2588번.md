## 💡 문제 이해

> 문제를 내가 이해한 방식으로 한 줄 요약

- **문제:**  
    세 자리 수 두 개가 주어질 때,  
    사람 손으로 곱셈할 때처럼  
    **각 자릿수 곱셈 결과 3개 + 전체 곱 1개**를  
    순서대로 출력하는 문제.
    

---

## 🚀 접근 방식 (Logic)

1. **입력 받기**
    
    - 첫 번째 줄: 세 자리 수 A (예: 472)
        
    - 두 번째 줄: 세 자리 수 B (예: 385)
        
    - 두 번째 입력은 `"3 8 5"`처럼 공백이 아닌 `"385"` 형태.
        
2. **자릿수 분리**
    
    - B의 자릿수를 직접 나누어야 한다:
        
        - 1의 자리: `B % 10`
            
        - 10의 자리: `(B / 10) % 10`
            
        - 100의 자리: `B / 100`
            
3. **중간 결과 계산**
    
    - A × B의 1의 자리
        
    - A × B의 10의 자리
        
    - A × B의 100의 자리
        
4. **마지막 결과**
    
    - A × B 전체 수
        
5. **출력**
    
    - 위 결과 4개를 줄바꿈하여 출력
        
6. **시간 복잡도**
    
    - 단순 연산 → `O(1)`

## 💻 정답 코드 (C#)

```C#
using System;

class Program
{
    static void Main()
    {
        int a = int.Parse(Console.ReadLine());
        int b = int.Parse(Console.ReadLine());

        int one = b % 10;       
        int ten = (b / 10) % 10; 
        int hun = b / 100;

        int A = a * one;
        int B = a * ten;
        int C = a * hun;
        int D = a * b;

        Console.WriteLine(A);
        Console.WriteLine(B);
        Console.WriteLine(C);
        Console.WriteLine(D);
    }
}
```

## 📝 오답 노트 & 회고

- **틀린 이유(초기 작성 코드):**
    
    - 두 번째 입력을 `"385"`로 받는데 `.Split()`을 사용해 `"3 8 5"`처럼 나누려 했음  
        → 입력 형식이 공백이 없어서 `Split()` 사용 불가.
        
    - 자릿수를 `input[0]`, `input[1]`, `input[2]` 로 접근하려 했지만  
        실제 입력은 배열이 아니라 문자열 1개였음.
        
- **배운 점:**
    
    - 공백 없는 입력을 다룰 때는  
        **문자열 인덱싱** 또는 **나머지/나눗셈 연산**으로 자릿수를 분리해야 한다.
        
    - 곱셈 문제에서도 “입력 형식”이 핵심이라는 것을 다시 확인함.
        
- **다른 방식 참고 포인트:**
    
    - 문자열로 받은 뒤 `bStr[2] - '0'` 방식으로 자릿수를 추출해도 된다.
        
    - 실제 수학 연산에서는 자릿수 분해가 기본이므로 연습하기 좋다.
