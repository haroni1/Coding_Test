## 💡 문제 이해

> 문제를 내가 이해한 방식으로 한 줄 요약

- **문제:**  
    공백 없이 주어진 N개의 숫자를 모두 더한 값을 출력하는 문제.
    
- **핵심 조건:**
    
    - 숫자는 문자열처럼 붙어서 입력된다.  
        예: `54321`, `7000000000000000000000000`
        
    - 이 숫자들을 하나씩 분리해 합을 구해야 한다.
        
    - 자리수가 너무 많을 수 있어서(최대 100자리) **정수로 파싱하면 안 됨!**
        

---

## 🚀 접근 방식 (Logic)

1. **입력 받기**
    
    - 첫 번째 줄: 숫자의 개수 N
        
    - 두 번째 줄: 공백 없이 이어진 숫자 문자열
        
2. **숫자 분해**
    
    - 문자열에서 각 문자를 `str[i]` 로 꺼낸 뒤  
        `'0'` 을 빼서 정수로 변환
        
3. **누적합 계산**
    
    - 반복문을 N번 돌면서 `sum += (str[i] - '0');`
        
4. **출력**
    
    - 계산된 합을 출력
        
5. **시간 복잡도**
    
    - 길이 N을 한 번 순회 → O(N), N ≤ 100 → 완전 가벼움
        

---

## 💻 정답 코드 (C#)

``` C#
using System;

class Program
{
    static void Main()
    {
        int n = int.Parse(Console.ReadLine());
        string str = Console.ReadLine();
        int add = 0;
        for(int i = 0; i<n; i++){
            add += str[i] - '0';
        }
        Console.WriteLine(add);
    }
}
```

## 📝 오답 노트 & 회고

- **틀린 이유(초기 접근):**
    
    - 입력 숫자가 매우 커질 수 있는데 `int.Parse()`로 받으려 해서 실패함  
        → 최대 100자리까지 가능하므로 정수형 변수(int, long)로는 담을 수 없다.
        
    - `str[i]` 를 사용하려고 `int` 타입 변수에 인덱싱을 시도하여 오류 발생.
        
- **배운 점:**
    
    - 붙어 있는 수열을 처리할 때는 **문자열 처리**가 기본이다.
        
    - 문자 `'5'` 를 숫자 5로 바꾸려면 `str[i] - '0'` 사용
        
    - 입력 형식(공백 여부, 자리수 크기)이 문제 풀이 방식에 큰 영향을 준다.
        
- **다른 방식 참고 포인트:**
    
    - `char.GetNumericValue(str[i])` 를 쓰는 방법도 있지만  
        `'0'` 을 빼는 방식이 더 빠르고 일반적이다.
        
    - C#의 `string` 은 배열처럼 `str[i]` 인덱싱이 가능하다는 점을 다시 기억하기.
