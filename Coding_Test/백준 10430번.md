## 💡 문제 이해

> 문제를 내가 이해한 방식으로 한 줄 요약

- **문제:**  
    세 정수 A, B, C가 주어질 때,  
    모듈러 연산에서 다음 네 가지 값을 각각 계산해 출력하는 문제.
    
- **핵심 조건:**
    
    - `(A + B) % C`
        
    - `((A % C) + (B % C)) % C`
        
    - `(A × B) % C`
        
    - `((A % C) × (B % C)) % C`
        
    - A, B, C는 2 이상 10000 이하의 자연수
        

---

## 🚀 접근 방식 (Logic)

1. **입력 받기**
    
    - `"A B C"` 형태로 들어오므로 `Split()` 사용
        
    - `int.Parse()` 로 정수로 변환
        
2. **모듈러 연산 성질 확인**
    
    - 덧셈
        
        `(A + B) % C == ((A % C) + (B % C)) % C`
        
    - 곱셈
        
        `(A × B) % C == ((A % C) × (B % C)) % C`
        
    - 문제는 이것을 실제 계산해 출력하도록 요구함
        
3. **출력**
    
    - 4개의 결과를 각 줄에 하나씩 출력
        
4. **시간 복잡도**
    
    - 단순 연산 → `O(1)`

## 💻 정답 코드 (C#)

```C# 
using System;

class Program
{
    static void Main()
    {
        string[] input = Console.ReadLine().Split();
        int a = int.Parse(input[0]);
        int b = int.Parse(input[1]);
        int c = int.Parse(input[2]);

        Console.WriteLine((a + b) % c);
        Console.WriteLine(( (a % c) + (b % c) ) % c);
        Console.WriteLine((a * b) % c);
        Console.WriteLine(( (a % c) * (b % c) ) % c);
    }
}
```

## 📝 오답 노트 & 회고

- **틀린 이유(초기 작성 코드):**
    
    - `A`, `B`, `C` 라는 **대문자 변수**를 사용했는데  
        실제 선언한 변수는 `a`, `b`, `c` (소문자)여서 컴파일 에러 발생함.
        
    - C# 은 **대소문자 구분 언어**임을 다시 확인해야 했다.
        
- **배운 점:**
    
    - 변수명을 정확하게 일치시켜야 하고  
        입력은 되도록 명확하게 소문자로 선언해 혼동을 피하는 것이 좋다.
        
    - 모듈러의 기본 성질 `(a + b) % c == ((a % c) + (b % c)) % c` 를 실습해볼 수 있었다.
        
- **다른 방식 참고 포인트:**
    
    - 실제 암호학·보안·해싱 등에서 모듈러 분배 법칙은 자주 쓰인다.
        
    - `(a * b) % c` 는 overflow 위험 때문에 미리 `% c` 하는 방식이 더 안전한 경우가 많다.
