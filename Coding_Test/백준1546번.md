## 💡 문제 이해

> 문제를 내가 이해한 방식으로 한 줄 요약

- **문제:**  
    학생의 점수들 중 _가장 높은 점수를 기준으로_ 모든 점수를 다시 환산한 후, 그 환산된 점수들의 **새로운 평균을 구하는 문제**
    
- **핵심 조건:**
    
    - 첫 줄: **과목 수 N**
        
    - 둘째 줄: **각 점수 입력 (공백 구분)**
        
    - 새 점수 공식 → `기존 점수 / 최고 점수 × 100`
        
    - 평균은 실수 형태(소수점 포함)로 출력해야 함
        

---

## 🚀 접근 방식 (Logic)

1. **입력 받기:**
    
    - `Console.ReadLine()` 으로 두 줄 입력 받음
        
    - 점수는 문자열 배열로 들어오기 때문에 바로 계산 불가 → 변환 필요
        
2. **자료 구조 선택:**
    
    - 점수를 저장할 `int[] scores` 배열 생성
        
    - 변환 전 점수를 그대로 저장해야 이후 반복에서 활용 가능
        
3. **최댓값 찾기:**
    
    - 반복문을 돌며 가장 큰 수를 `max` 변수에 저장
        
    - 새로운 점수 계산 시 기준이 되는 값
        
4. **새로운 점수 계산 & 총합 구하기:**
    
    - `(double)score / max * 100`으로 계산
        
    - 누적 합계를 `sum`에 저장
        
5. **평균 계산:**
    
    - `sum / n`
        
    - double 로 계산되기 때문에 자동으로 소수 유지됨
        
6. **시간 복잡도:**
    
    - 배열 두 번 순회 → `O(N)`
        
    - N ≤ 1000 이므로 매우 빠름

## 💻 정답 코드 (C#)

```C#
using System;

class Program
{
    public static void Main()
    {
        int n = int.Parse(Console.ReadLine());
        string[] arr = Console.ReadLine().Split();

        int[] scores = new int[n];

        int max = 0;
        double sum = 0;

        // 점수 배열 생성 & 최대값 찾기
        for (int i = 0; i < n; i++)
        {
            scores[i] = int.Parse(arr[i]);

            if (scores[i] > max)
            {
                max = scores[i];
            }
        }

        // 새로운 점수 계산
        for (int i = 0; i < n; i++)
        {
            sum += (double)scores[i] / max * 100;
        }

        double average = sum / n;
        Console.WriteLine(average);
    }
}
```

## 📝 오답 노트 & 회고

- **틀린 이유(초반 생각):**
    
    - 처음엔 "점수 합 / max ×100" 하면 끝일 줄 알았지만  
        문제는 **각 점수를 변환 후 평균**을 구해야 해서 계산 방식이 달랐다.
        
- **배운 점:**
    
    - 문제 설명에 있는 수식이 전체에 적용되는지,  
        **각 요소에 개별 적용되는지 반드시 구분해야 한다.**
        
    - `sum +=` 은 누적이고, `sum =` 은 덮어쓰기라는 차이를 제대로 이해했다.
        
    - 실수 계산이 필요한 경우 `double` 캐스팅이 중요하다.
        
- **다른 방식 참고 포인트:**
    
    - `scores.Max()` (LINQ)로 더 깔끔하게 구현 가능하지만  
        반복문으로 직접 구현한 게 **입력 처리와 배열 활용 이해에 더 도움이 됐다.**
