## 💡 문제 이해

> 문제를 내가 이해한 방식으로 한 줄 요약

- **문제:**  
    두 정수 `A`와 `B`가 주어졌을 때, **A ÷ B 의 값을 소수점 형태로 출력하는 문제**
    

---

## 🚀 접근 방식 (Logic)

1. **입력 받기**
    
    - 한 줄에 `"A B"` 형태로 들어오기 때문에 `Split()`을 사용해 분리한다.
        
2. **자료형 변환**
    
    - 나눗셈 결과가 소수가 될 수 있기 때문에 `int`가 아닌 `double` 형식으로 변환한다.
        
3. **계산**
    
    - 단순한 나눗셈 연산: `a / b`
        
4. **출력**
    
    - 결과를 `Console.WriteLine()`으로 출력한다.
        
    - 이 문제는 **절대/상대 오차 10^-9 이하**면 정답 처리되므로 형식 지정 없이 출력해도 된다.
        
5. **시간 복잡도**
    
    - 연산 1번 → `O(1)`
        

---

## 💻 정답 코드 (C#)

```C#
using System;

class Program
{
    static void Main()
    {
        string[] input = Console.ReadLine().Split();
        double a = double.Parse(input[0]);
        double b = double.Parse(input[1]);

        Console.WriteLine(a / b);
    }
}

```

## 📝 오답 노트 & 회고

- **틀린 이유(초반 생각):**
    
    - 처음에는 `int`로 변수 선언해 계산했지만,  
        정수 나눗셈은 결과가 정수로 떨어지기 때문에 원하는 출력 형태가 나오지 않았다.
        
- **배운 점:**
    
    - **나눗셈 문제에서는 결과가 소수 형태가 될 수 있으므로**  
        입력 받는 값 및 계산식을 반드시 `double`이나 `float`으로 처리해야 한다.
        
    - 백준 문제에서 “오차 범위 허용” 문구가 있으면  
        실수 연산이 필요하다는 힌트라는 걸 깨달았다.
        
- **다른 방식 참고 포인트:**
    
    - 출력 형식 지정 없이도 정답 처리가 되지만,  
        상황에 따라 `string.Format("{0:F10}", a/b)` 같은 형식 제어도 활용 가능하다.
